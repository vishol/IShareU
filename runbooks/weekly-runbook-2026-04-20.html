Application Overview

The application is a Java 17 and Spring Boot 3.2 service that combines user authentication, file-sharing workflows, and operational dashboards in a single deployable unit. Thymeleaf templates render the user experience, while Spring MVC controllers route upload, download, dashboard, and admin traffic.

Files are stored in Supabase Storage, while file metadata and user records are stored in PostgreSQL. Download flows are governed by unique links, expiration rules, maximum download counts, and activity status checks.

Primary Entry Points/, /login, /signup, /dashboard, /file/upload, /file/download/{uniqueLink}, /admin/*
Authentication ModelForm login via Spring Security with role-based admin protection.
Storage PatternBinary content in Supabase bucket, relational metadata in PostgreSQL, and health/metrics via Actuator.
Operational GuardrailsPer-IP rate limits, file-type allow-list, size limits, link expiration, and download-count enforcement.

Processing Steps

  1. User authenticates through Spring Security and reaches the upload workflow.
  2. FileController validates request rate, file presence, allowed type, filename sanity, and 1 GB size ceiling.
  3. SupabaseStorageService uploads the binary to the configured Supabase bucket using a generated unique filename.
  4. FileService stores file metadata, unique link, expiry, download limits, and ownership in PostgreSQL.
  5. The UI returns a success page with the generated download URL.
  6. For downloads, the application validates link activity, expiry window, and max-download state before generating a signed Supabase URL.
  7. The download count is incremented and the browser is redirected to the signed object URL.
  8. Admin pages aggregate user and file statistics, paginate lists, and support delete actions for operational cleanup.
